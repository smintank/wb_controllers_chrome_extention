# PRD: Улучшение popup Chrome-расширения Wiren Board Tracker

## Problem Statement

Плагин `Wiren Board Tracker` уже умеет автоматически запоминать посещенные контроллеры и показывать их в popup, но текущее поведение недоделано и ненадежно. Пользователь не может быстро перейти по SSH, строка контроллера содержит конфликтующие действия, статус доступности работает неточно, а удаление контроллера занимает основной слот действий. В результате popup не выполняет роль удобной панели быстрого доступа к локальным контроллерам Wiren Board.

## Solution

Расширение должно превратить popup в компактный launcher для обнаруженных контроллеров Wiren Board. Пользователь открывает popup и сразу видит список контроллеров, отсортированный по serial. Клик по строке открывает web UI контроллера по `.local`-адресу. Отдельная SVG-иконка `SSH` в строке открывает `ssh://root@wirenboard-<serial>.local`. Дополнительные действия, включая удаление контроллера, скрываются в `⋯`-меню. Статус каждого контроллера должен иметь три состояния: серый `checking`, зеленый `online`, красный `offline`. При включенном фильтре `Только онлайн` серые строки временно остаются видимыми до завершения проверки, после чего красные скрываются.

## User Stories

1. Как пользователь Wiren Board, я хочу видеть все ранее обнаруженные локальные контроллеры в одном popup, чтобы быстро открывать их без ручного ввода адресов.
2. Как пользователь Wiren Board, я хочу, чтобы клик по строке контроллера открывал его web UI, чтобы основное действие было быстрым и очевидным.
3. Как пользователь Wiren Board, я хочу отдельное действие для SSH в каждой строке, чтобы открывать терминальный доступ без ручного ввода команды.
4. Как пользователь Wiren Board, я хочу, чтобы SSH-действие использовало локальный `.local`-хост, чтобы web- и SSH-доступ были согласованы.
5. Как пользователь Wiren Board, я хочу, чтобы автодобавление контроллеров продолжало работать по известным шаблонам Wiren Board URL, чтобы список пополнялся автоматически во время обычной работы в браузере.
6. Как пользователь Wiren Board, я хочу, чтобы цвета статуса различали состояния проверки, доступности и недоступности устройства, чтобы было понятно, статус уже известен или еще уточняется.
7. Как пользователь Wiren Board, я хочу, чтобы popup отрисовывался сразу, а статус обновлялся асинхронно, чтобы интерфейс оставался отзывчивым даже при нескольких контроллерах.
8. Как пользователь Wiren Board, я хочу, чтобы фильтр `Только онлайн` не скрывал слишком рано устройства, которые еще проверяются, чтобы список не мигал и не выглядел пустым по ошибке.
9. Как пользователь Wiren Board, я хочу, чтобы destructive-действия были скрыты в дополнительном меню, чтобы основной список был сфокусирован на переходах.
10. Как пользователь Wiren Board, я хочу удалять контроллер из списка через меню строки, чтобы поддерживать список в актуальном состоянии.
11. Как пользователь Wiren Board, я хочу видеть понятное пустое состояние с подсказкой, как добавить первый контроллер, чтобы понимать, что достаточно открыть его web UI.
12. Как разработчик, я хочу разделить логику popup на более понятные ответственности, чтобы UI-действия, хранение устройств и обновление статусов можно было развивать без повторных поломок.
13. Как разработчик, я хочу избавиться от недоделанных полей и неоднозначной click-модели в popup, чтобы новые функции не наслаивались на сломанную основу.

## Implementation Decisions

- Канонической идентичностью контроллера остается serial, извлекаемый из поддерживаемых шаблонов URL Wiren Board.
- Контроллер хранится и открывается только по `.local`-хосту вида `wirenboard-<serial>.local`.
- Удаленные Wiren Board URL продолжают использоваться только как источник автообнаружения, но не как целевой адрес перехода.
- Автообнаружение сохраняется; ручное добавление контроллера в эту итерацию не входит.
- Основная подпись строки в списке отображает только serial; пользовательские alias и редактируемые имена не добавляются.
- Строка popup перестраивается в три независимые зоны взаимодействия: основная область строки для HTTP, отдельное действие SSH и отдельный триггер overflow-меню.
- Текущий подход с overlay-ссылкой поверх строки заменяется на структуру, где secondary actions не конфликтуют с основным переходом.
- Для SSH добавляется отдельное действие, открывающее `ssh://root@wirenboard-<serial>.local`.
- Для SSH используется локальный SVG-asset внутри проекта; внешние icon-библиотеки не подключаются.
- Удаление переносится в `⋯`-меню строки и выполняется без подтверждения.
- После запуска web UI или SSH popup закрывается стандартным образом, как обычный Chrome popup.
- Модель локализации RU/EN сохраняется, а новые элементы popup должны быть согласованы с существующей схемой переводов.
- Бинарная модель статуса заменяется на явную трехсостоячную: `checking`, `online`, `offline`.
- При открытии popup список отрисовывается сразу из сохраненных данных, после чего асинхронно выполняются HTTP-проверки доступности, а строки обновляются по мере получения результата.
- Статус `online` определяется исключительно доступностью HTTP web UI, а не доступностью SSH.
- Фильтр `Только онлайн` применяется к итоговому статусу, но строки в состоянии `checking` остаются видимыми до завершения проверки.
- Порядок списка стабилен и определяется serial, без перестановки элементов при изменении статуса.
- Автообнаружение остается ограниченным текущими поддерживаемыми шаблонами URL Wiren Board, чтобы не засорять storage ложными совпадениями.
- Недоделанные зависимости popup от отсутствующих полей должны быть устранены как часть этой работы.
- В этой итерации не расширяется модель данных: без ручного добавления, без alias, без выбора между локальным и удаленным адресом.

## Testing Decisions

- Хороший тест подтверждает три основных пользовательских контракта: клик по строке открывает HTTP, иконка SSH открывает SSH URI, действие из overflow-меню удаляет контроллер.
- Хороший тест подтверждает детерминированные переходы состояний: начальное `checking`, затем `online` или `offline`, а также корректную работу фильтрации.
- Логику автообнаружения и извлечения serial нужно тестировать отдельно от popup UI, особенно для поддерживаемых шаблонов хостнеймов Wiren Board.
- Логику рендера popup стоит тестировать на уровне формирования строк, привязки действий, сортировки по serial и отображения пустого состояния.
- Удаление должно тестироваться на корректное удаление нужного контроллера из storage через действие overflow-меню.
- Фильтр должен тестироваться на смешанном наборе устройств в состояниях `checking`, `online` и `offline`, чтобы проверить точные правила видимости.
- Приоритет нужно отдавать тестам для чистых функций парсинга, подготовки данных для UI и обработки состояния строк, потому что это самые устойчивые и тестируемые интерфейсы этого расширения.
- В текущей кодовой базе, судя по локальному контексту, готовых тестов нет, поэтому новая работа должна заодно задать базовый паттерн тестирования.

## Out of Scope

- Ручное добавление контроллера.
- Редактируемые имена или alias контроллеров.
- Использование удаленных Wiren Board доменов как целевых адресов перехода.
- Tooltip-подсказки для SSH и overflow-меню.
- Подтверждение перед удалением.
- Отдельный экран настроек или управления списком.
- Проверка доступности по SSH.
- Расширение эвристик автообнаружения за пределы текущих Wiren Board шаблонов URL.
- Изменения брендинга расширения, кроме новой SVG-иконки SSH и обновленного layout строки.

## Further Notes

- Расширение достаточно компактное, поэтому эта работа должна стать точкой, в которой popup-логика разделяется на более понятные модули вместо дальнейшего роста одного связанного скрипта.
- Главный технический риск связан с текущим способом проверки HTTP-доступности, который уже сейчас ненадежен как источник правдивого online/offline-статуса.
- Второй риск связан с текущей click-архитектурой строки popup: добавление SSH и overflow поверх существующего overlay-подхода почти гарантированно приведет к конфликтам событий, если структуру строки не пересобрать.
- Третий риск связан с недоделанным popup-кодом, который ссылается на отсутствующие поля и должен быть очищен в рамках этой задачи, а не сохранен как есть.
